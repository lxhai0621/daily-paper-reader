# Evaluating the Search Agent in a Parallel World 

Jiawei Chen Xintian Shen Lihao Zheng Lifu Mu Haoyi Sun Ning Mao Hao Ma ∗ Tao Wei ∗ Pan Zhou † Kun Zhan †

Integrating web search tools has significantly extended the capability of Large Language Models (LLMs) to address open-world, real-time, and long-tail problems. However, evaluating these Search Agents presents formidable challenges. First, constructing high-quality deep search benchmarks is prohibitively expensive, while unverified synthetic data often suffers from unreliable sources. Second, static benchmarks face dynamic obsolescence: as internet information evolves, complex queries requiring "deep research" often degrade into simple retrieval tasks due to increased popularity (difficulty collapse), and ground truths become outdated due to temporal shifts (fact drift). Third, attribution ambiguity confounds evaluation, as an agent’s performance is often dominated by its parametric memory rather than its actual search and reasoning capabilities. Finally, reliance on specific commercial search engines introduces variability that hampers reproducibility. To address these issues, we propose a novel framework–Mind-ParaWorld (MPW) for evaluating Search Agents in a "Parallel World." Specifically, MPW samples real-world entity names to synthesize "future scenarios & questions" situated beyond the model’s knowledge cutoff. A "ParaWorld Law Model" then constructs a set of indivisible Atomic Facts and a unique ground-truth answer for each question. During evaluation, instead of retrieving real-world results, the agent interacts with a ParaWorld Engine Model (PEM) that dynamically generates SERP-style evidence (news snippets) grounded in these inviolable Atomic Facts. We release MPW-Bench, an interactive benchmark spanning 19 domains with 1,608 instances. Experiments across three evaluation settings show that, while search agents are strong at evidence synthesis given complete information, their performance is limited not only by evidence collection and coverage in unfamiliar search environments, but also by unreliable evidence sufficiency judgment and when-to-stop decisions—bottlenecks. ## 1 Introduction 

Large Language Models (LLMs) [ 14 , 1, 7, 27 , 5 , 21 , 13 ] have revolutionized information processing but remain constrained by two fundamental limitations: temporal staticity, where fixed parameters impose a strict knowledge cutoff, and long-tail deficits, which hinder performance on obscure or specialized queries. While Retrieval-Augmented Generation (RAG) [ 9] systems and workflow paradigm [ 16 , 10 ] offer a partial remedy, they are typically limited to single-step retrieval and lack the agency to refine their search scope. In contrast, Search Agents [ 11 , 25 , 3] operating under the

ReAct [ 30 ] paradigm represent a significant leap forward. These agents function as autonomous, iterative systems capable of problem decomposition and strategic planning. By dynamically adjusting their search trajectories based on intermediate feedback, Search Agents can synthesize multi-source information to solve "Deep Search" problems that are intractable via simple, direct queries. Despite the rapid evolution of Search Agents, evaluation methodologies have failed to keep up with the times. Traditional benchmarks [ 4] measuring isolated fact retrieval, such as SimpleQA [ 23 ], have become saturated by models equipped with basic browsing capabilities. Consequently, current approaches [ 26 ] to evaluating "Deep Search" capabilities—primarily relying on static datasets—face three fundamental challenges: 

1. Dynamic Obsolescence of Static Benchmarks: The real-world search environment is highly fluid. As information accumulates on the web, complex problems requiring multi-hop reasoning often degrade into trivial retrieval tasks due to the emergence of direct sources—a phenomenon we term "Difficulty Collapse." Furthermore, "Fact Drift" (e.g., changes in sports records, stock rankings, or pricing) renders the ground truth of static benchmarks rapidly outdated. 

2. Attribution Ambiguity: The distinction between parametric memory and genuine agentic reasoning remains a critical scientific hurdle. Given the vast pre-training corpora of LLMs, it is often unclear whether a model solves a problem through tool usage or merely by reciting memorized long-tail knowledge. When tools fail to provide all necessary clues, evaluation degenerates into a test of knowledge retention rather than search capability. 

3. The Cost-Quality Paradox: Constructing high-quality deep search benchmarks requires pro-hibitive expert effort, limiting scalability. In contrast, fully automated pipelines often yield synthetic data plagued by logical loopholes or unreliable sources, compromising robustness. Moreover, reliance on commercial search engines introduces uncontrollable engine bias via opaque ranking algorithms, creating an unfair comparison environment. To address these challenges, we propose Mind-ParaWorld (MPW), an evaluation framework that constructs a dynamic “parallel world” cognitively isolated from the agent’s internal knowledge. Figure 1 provides an overview of the MPW pipeline. Specifically, MPW samples real-world entity names and uses a Parallel World Model to extrapolate future-situated scenarios from entity pairs, formulating them as questions beyond the model’s knowledge cutoff. A ParaWorld Law Model then decomposes each question into a set of inviolable, indivisible Atomic Facts (the “physical laws” of the scenario) and derives a unique ground-truth answer under these laws. This design ensures that retrievable information is logically closed and factually consistent, mitigating the inconsistencies common in uncontrolled synthetic benchmarks. In MPW, the real-world search engine is replaced by a ParaWorld Engine Model (PEM). Instead of retrieving static web pages, the agent interacts with PEM, which dynamically generates news snippets grounded in the scenario’s Atomic Facts. Crucially, MPW enforces an anti-shortcut mechanism: evidence availability is conditioned on query granularity, suppressing broad, underspecified queries and rewarding decomposed, fine-grained queries with fact-grounded evidence, thereby reducing shortcut-induced confounding and ensuring that the evaluation signal reflects decomposition and evidence-seeking. Building on the proposed MPW method, we further introduce a novel evaluation benchmark, MPW-Bench. MPW-Bench adopts real-world entity names as construction anchors and curates entity pools spanning 19 domains. Within each domain, entity pairs are deliberately constructed through controlled sampling, systematically injecting compositional diversity while preserving contextual realism and domain coherence. This pairing strategy naturally gives rise to information need patterns prevalent in deep search tasks—including comparison, filtering, aggregative computation, and rule-specification judgment. We then employ the Parallel World Model to generate a series of counterfactual-scenario questions and subject all samples to automated consistency and credibility auditing. The resulting benchmark, MPW-Bench, comprises 1,608 instances spanning multiple levels of complexity. ## 2 The Mind-ParaWorld Framework 

2.1 Definition of the Four Stages of Agent Development 

To clarify evaluation objectives, in this paper, we define the capability evolution of Tool Integrated Reasoning (TIR) agents as four phases: 

Phase 1 (Tool Execution Syntax): The agent knows how to invoke tools with correct syntax. 

Phase 2 (Selection): The agent knows whether to invoke tools for a task, performs simple multi-step tool use, and engages in multi-turn reasoning incorporating tool-call results. 

Phase 3 (Decomposition & Planning): When facing complex problems, the agent knows how to decompose the problem, perform reasoning and action planning, and translate this understanding into precise sequences of tool calls. 

Phase 4 (Adaptation & Correction): When the optimal action and reasoning path is blocked (e.g., the environment does not return expected information), the agent can correct the path based on environmental feedback and find alternative solutions to solve the problem (Resilience Error Handling). MPW is specifically designed to evaluate Search Agents equipped with the web search tool at Phase 3 and Phase 4. 

2.2 Overview of Mind-ParaWorld Framework 

The core idea of MPW is to construct a controllable parallel-world environment that is cognitively isolated from a model’s parametric memory, thereby enabling a cleaner evaluation of Search Agents. 3As illustrated in Figure 1, MPW consists of three stages: (1) parallel-world question construction, (2) parallel-world law construction, and (3) agent–environment interaction for evaluation. Specifically, in the first stage, a Parallel World Model generates future-situated questions grounded in sampled real-world entities. In the second stage, a ParaWorld Law Model decomposes each question into a set of inviolable Atomic Facts and derives a unique ground-truth answer under these laws. In the third stage, a ParaWorld Engine Model provides a unified web-search interface and returns fact-grounded evidence as environment feedback, through which the agent iteratively collects evidence and produces the final answer. We next describe how MPW constructs ParaWorld questions and the underlying principles that guide their design. 

2.3 Construction of the ParaWorld Questions 

MPW evaluates Search Agents primarily at Phase 3 and Phase 4. Its goal is not to test whether an agent can retrieve a single isolated fact, but whether it can, given a complex question, identify the required information needs, issue actionable queries, and iteratively refine its search trajectory through multi-turn interaction to assemble evidence and synthesize a final answer. The design of ParaWorld questions follows three principles: multi-fact dependency, parametric-memory isolation, and anti-shortcut. 

Multi-fact dependency. Deep-search problems in real settings typically involve multiple constraints (e.g., time, location, measurement conventions, target entities, and conditional filters) and require aggregating evidence from multiple sources. Motivated by this observation, MPW designs ParaWorld questions to inherently require multiple facts: each question is associated with an implicit fact-dependency graph, and the agent must identify and verify multiple information nodes via multi-turn search before it can complete the final computation or decision. 

Parametric-memory isolation. ParaWorld questions place the key facts in a future time range beyond the model’s knowledge cutoff. The purpose is to ensure that, structurally, the answer cannot be derived from prior knowledge alone: without using tools to progressively verify intermediate facts, the model cannot infer the correct answer. In other words, the design targets a “must-search” solution path, rather than merely creating unfamiliarity by shifting timestamps. 

Anti-shortcut. MPW adopts an anti-shortcut principle: while a question may be complex in surface form, its solvable path should decompose into multiple actionable and locatable search sub-goals. As a result, high-quality queries correspond to the precise retrieval of specific sub-information, rather than issuing the original question as a single monolithic query. This principle ensures that the downstream evaluation results primarily reflect the search agent’s decomposition, planning, and evidence synthesis, rather than whether a search engine happens to return an aggregated SERP containing answers directly. Based on these principles, we employ a Parallel World Model to construct ParaWorld questions. As shown in Figure 1, for a given set of related entity pairs, the model generates multiple questions with increasing levels of difficulty. The difficulty increase is reflected in the structural complexity of the underlying task graph: from cases where sub-goals are explicit, and subproblems are parallel and largely independent, to cases where sub-goals are implicit and involve multi-hop dependencies in which later sub-queries depend on intermediate results; meanwhile, the task objective evolves from a single overall goal to multiple goals and multiple required outputs. 

2.4 Constructions of the ParaWorld Laws 

To ensure that the parallel-world evaluation environment is controllable and reproducible, MPW constructs a set of ParaWorld Laws for each ParaWorld question. We define these laws as a collection of inviolable Atomic Facts that jointly serve as the “physical laws” of the parallel world. During evaluation, all information synthesized by the environment must be consistent with these laws, and the ground-truth answer to the question must be uniquely determined under them. 

Representation of Atomic Facts. As shown in Figure 1, in MPW, Atomic Facts are stored as key–value pairs in the form of atomic query description → atomic answer , where the key is a natural-language description of a minimal information unit, and the value is its unique instantiation 4in the parallel world (e.g., a number, an entity assignment, a Boolean judgment, or a short text span). We additionally allow a special type of Atomic Fact that specifies computation or convention rules .For example, a question may require aggregation under a particular seat-count basis, discount rule, or compliance convention; since such rules are indispensable for solving the task, they are treated as part of the ParaWorld Laws and stored alongside other Atomic Facts. The construction of ParaWorld Laws follows three core objectives: (1) Consistency: Atomic Facts must not conflict with one another. (2) Atomicity & Sufficiency: each Atomic Fact should correspond to a minimal, independently retrievable information unit (i.e., it cannot be further decomposed into smaller searchable facts). Meanwhile, the fact set should be sufficient to determine the answer, covering all information required for solving the question, including necessary computation or convention rules. (3) Uniqueness: given the set of Atomic Facts and their implied constraints, the question should admit a unique ground-truth answer, ensuring that evaluation remains well-defined. Based on these objectives, given a ParaWorld question, we use a ParaWorld Law Model to analyze the question, identify the key intermediate variables and information dependencies required for solving it, and instantiate them as a set of Atomic Facts (key–value pairs). This process does not aim to reproduce any single fixed decomposition path; rather, it extracts the lowest-level truth anchors needed to answer the question, such that a Search Agent can progressively hit them through appropriate queries during interaction. After generating the Atomic Facts, the ParaWorld Law Model performs explicit logical reasoning or numerical computation under its own specified rules to derive the final ground-truth answer, and then self-checks the generated facts for coherence and non-decomposability. 

2.5 Generation of the ParaWorld News 

During MPW evaluation, we replace the real-world search engine with a ParaWorld Engine Model 

(PEM), which provides a unified web_search tool interface for Search Agents. Given the original question, the corresponding ParaWorld Laws (i.e., the set of Atomic Facts), and the agent’s current query, PEM generates a simulated SERP, where each entry contains a title, a content snippet, and a timestamp. All information generated by PEM must strictly use the Atomic Facts as the sole source of truth, thereby ensuring that environment feedback is both consistent and controllable. 

Query type classification: atomic vs. compound queries. As shown in Figure 1, PEM first performs a structural classification of the query, assigning it to either an Atomic Query or a Compound Query . Here, “atomic” means that the query is intended to locate one minimal information unit (typically anchored on a single entity and a single attribute/relation), whereas compound queries bundle multiple entities/attributes or express comparison/aggregation/analysis intent that cannot be mapped to a single retrievable fact. Importantly, this classification depends only on the query’s structure and intent, and is independent of whether a matching fact exists in the ParaWorld Laws; in particular, an atomic query that misses all facts should not be treated as compound. If the query is classified as atomic, PEM further checks whether it can be matched to a single Atomic Fact in the ParaWorld Laws: a successful match yields hit=1 and returns the corresponding fact key (as matched_fact_keys , with at most one element), while a failed match yields hit=0 . If the query is classified as compound or low-quality, PEM enforces hit=0 . During evaluation, PEM returns a hit log along with the SERP, which is used for downstream process-level metrics and behavioral diagnostics. PEM always produces a fixed number of search results (four entries), while controlling the amount of actionable information based on availability: (1) For atomic queries with hit=1 , PEM generates four results related to the query and guarantees that at least one “truth entry” explicitly contains the value of the matched Atomic Fact; the remaining entries provide relevant background but must not introduce any facts that contradict the ParaWorld Laws. (2) For atomic queries with hit=0 , PEM still returns four topically related background entries, but must not reveal any concrete values from the Atomic Facts. (3) For compound/low-quality queries, PEM returns four high-entropy noisy entries consisting of superficially relevant commentary, speculation, or peripheral information, and explicitly avoids leaking any Atomic Fact values or the final answer. This combination of query gating and controlled information provision constitutes MPW’s anti-shortcut mechanism: strategies that directly issue the original question as a single query, or overload one query with multiple subgoals, are unlikely to obtain actionable evidence. ## 3 The Construction of the MPW-Bench 

3.1 Collection of the Real-world entities and In-domain Entity Pairing and Sampling 

MPW-Bench is grounded in real-world entities to ensure realistic contexts and broad domain coverage. We curate an entity pool spanning 19 domains (as listed in Table 2), including sports, entertainment, consumer products, and technology. This entity pool provides topical boundaries and semantic constraints for downstream ParaWorld question generation, allowing questions to naturally take the form of comparisons, conditional filtering, numerical aggregation, and compliance-style decision making that commonly arise in deep-search scenarios. To introduce compositional diversity while keeping the domain context fixed, we only form in-domain 

entity pairs by pairing entities within the same domain, and use these pairs as seeds for question construction. Such entity pairing encourages multi-fact, constraint-based information needs (e.g., comparisons under specific conditions) rather than simple single-query fact lookup. For each entity pair, we aim to generate four questions using the Parallel World Model. We first sample a total of 500 entity pairs. To achieve balanced coverage across the 19 domains, we sample uniformly from each domain-specific entity pool; when a domain contains fewer entities than required for uniform sampling, we compensate by incrementally sampling additional pairs from other domains in a balanced manner. Within each domain, entity pairs are selected via random sampling. 

3.2 Construction of Questions, Laws and Ground-truth 

For each sampled entity pair, we generate four ParaWorld questions. For every question, we then construct the corresponding ParaWorld Laws as a set of Atomic Facts (which may include rule/con-vention facts such as metric definitions or aggregation conventions), and derive a unique ground-truth answer under these laws. Each scenario is thus finalized in the form of question–laws–answer , and can be directly used for offline, interactive evaluation. Applying this pipeline, we obtain a stage-0 version of MPW-Bench consisting of 19 domains with 2,000 questions, each paired with its Atomic Facts and ground truth. We then perform automated quality auditing over the entire stage-0 benchmark using Gemini 3 Pro in two steps. In the first step, the auditor checks that (i) the ParaWorld Laws are internally non-contradictory, and (ii) the laws satisfy basic plausibility constraints grounded in real-world commonsense (e.g., realistic travel times and valid units). In the second step, the auditor is provided with the full set of Atomic Facts and asked to answer the question; the resulting answer is compared against the provided ground truth for consistency. We repeat this second-step check three times to reduce stochasticity. This automated auditing filters out 392 instances (19.6%), yielding the final MPW-Bench with 1,608 instances. We assign three complexity tiers post-hoc based on the number of atomic facts |F| , and report the resulting distribution in Table 1. 3.3 Data Format and Benchmark Interface 

MPW-Bench is constructed as a collection of interactive scenarios . Each scenario instantiates a ParaWorld question together with its ParaWorld Laws (Atomic Facts) and a unique ground-truth answer, enabling offline evaluation under a controlled search environment. 

Data representation. Each instance contains (i) a future-situated question, (ii) a set of Atomic Facts expressed as natural-language key–value pairs (including optional rule/convention facts such as metric definitions), and (iii) the derived ground-truth answer. The data sample is shown in the appendix. 

Interactive protocol. Evaluation is conducted with a single tool, web_search . A PEM implements this tool by producing a simulated SERP for each agent query, strictly grounded in the Atomic Facts of the current scenario. In addition to the SERP, PEM outputs a hit log indicating whether the query is considered compound/low-quality and whether it matches an atomic fact, enabling process-level analysis of agent behaviors such as fact coverage and query efficiency without altering the agent’s interaction interface. In our implementation, PEM returns a fixed-size SERP with four entries per query and logs at most one matched atomic-fact key for each tool call. 

## 4 Experiments 

4.1 Metrics for MPW-Bench 

In MPW, we primarily evaluate search agents on MPW-Bench using pass@1 . In addition, we introduce several process-aware metrics to characterize an agent’s overall performance. 

Fact Coverage Rate (FCR). To measure whether an agent retrieves sufficient evidence, we define the Fact Coverage Rate . 4.2 Application Details 

To better interpret the performance of Search Agents on MPW-Bench, we design three evaluation settings of increasing difficulty, which respectively characterize (i) an upper bound of evidence synthesis, (ii) decomposition and evidence coverage under guided querying, and (iii) fully end-to-end capability spanning decomposition, query strategy, coverage, and synthesis. For the agent mode, we evaluate all models using the ReAct paradigm. Specifically, Setting A (Oracle-Facts QA, upper bound) directly provides the model with the original question and the complete set of atomic facts for that instance (including necessary com-putation/convention rules), and forbids any tool calls. This setting evaluates the model’s ability to 7integrate information and synthesize the final answer given all required evidence and conditions, serving as an achievable upper bound once all necessary information has been obtained through search. 

Setting B (Guided Search) retains the end-to-end interactive form: the model still needs to decom-pose the problem and interact with PEM via web_search to collect evidence, but we explicitly guide the model through the system prompt to construct “hittable” atomic queries (e.g., single entity + single attribute), so as to reduce confounding from query formulation quality. To differentiate between two guidance strengths—lightweight rule prompting versus stronger decomposition exemplars—Setting B includes two prompt variants: the Guidance Prompt provides a concise guide for query construction and query decomposition examples, while the Few-shot Decomposition Prompt not only includes multiple query decomposition examples, but also provides planning processes for solving questions from multiple cases as reference. 

Setting C (End-to-End Search) provides only the original question and basic tool instructions, without any additional query guidance. The model must independently decompose the problem, formulate queries, infer how to construct high-quality queries from interaction, retrieve sufficient evidence coverage, and synthesize the final answer. More detailed explanations and the exact prompts for Settings A/B/C are provided in the appendix. To ensure fair comparison, we apply the same termination conditions to all models: inference stops immediately when the agent outputs <answer>... </answer> or when the maximum interaction budget of 32 turns is reached. In our experiments, we use Qwen3-235B-A22B-Thinking as the PEM. For correctness evaluation, we adopt an LLM-as-Judge protocol, the judge model is also Qwen3-235B-A22B-Thinking and the system prompt is provided in the appendix. 

4.3 Setting A Results: Evidence-synthesis upper bound 

Table 3 reports results under Setting A (Oracle-Facts QA) . Setting A estimates an upper bound on performance once all necessary evidence has been acquired, serving both to validate task solvability and to characterize evidence synthesis and aggregation ability under complete information. Most models achieve strong Pass@1 under Setting A (67.23%–91.04%). Qwen3-32B attains the best overall performance (91.04% overall; 94.86% on Easy; 85.75% on Hard), while GLM-4.7 is also competitive (89.80% overall). MindWatcher 32B, Qwen3-30B-A3B, and Tongyi DeepResearch all exceed 83% overall. These results indicate that, given complete evidence, modern models are generally capable of integrating the provided information and synthesizing correct answers, supporting that MPW-Bench is well-posed under atomic-fact constraints. We further observe performance degradation with increasing complexity for most models (e.g., Qwen3-32B drops from 94.86% in Easy to 85.75% in Hard), suggesting that even without retrieval, harder instances require non-trivial multi-condition aggregation, correct application of convention-s/rules, and longer-chain reasoning. Importantly, the high accuracy in Setting A provides a clear reference point for the end-to-end evaluations in Settings B/C: when performance in these settings falls far below the Setting A upper bound, the dominant bottleneck is more plausibly evidence acquisition and coverage through search, rather than an inability to reason given complete evidence.

Fewshot vs. Guidance. The gains from the two system prompts are not uniform across models, exhibiting clear model dependence. Meanwhile, stronger agents such as MindWatcher and MiniMax-m2/m2.1 typically increase their ToolCalls with difficulty, indicating that they adjust search budget in response to task complexity. Nevertheless, increased tool usage does not fully offset the drop in coverage. This provides additional evidence that MPW-Bench’s complexity tiers reliably scale task difficulty and place increasing demands on search planning and evidence coverage. 

Tool usage and stopping behavior. ToolCalls further reveal substantial differences in interaction strategies under guided search. MindWatcher 32B typically issues more queries (about 10 on Overall and over 14 on Hard), accompanied by higher Hit Rate and FCR. In contrast, Qwen3-32B achieves relatively competitive accuracy with far fewer tool calls (about 2.6 on Overall), suggesting higher query efficiency. However, a broader pattern emerges: many models output a final answer before achieving sufficient evidence coverage, exhibiting a clear tendency of premature stopping , even though the system prompt explicitly provides hittable query-construction rules. ## 5 Related Work 

5.1 Search Agents 

Integrating web search tools has substantially improved the ability of LLMs to address real-time and long-tail queries. However, the standard RAG [ 9 ] paradigm is typically limited to one-shot retrieval and does not enable models to actively refine information needs or adapt search strategies based on intermediate evidence. The introduction of the ReAct [ 30 ] paradigm enabled agents to interleave reasoning and tool use, allowing them to decompose complex questions into sub-goals, plan multi-step search trajectories, and iteratively adjust actions according to retrieved feedback before synthesizing an answer from multiple sources. As one of the first tool-integrated reasoning agents deployed at scale, OpenAI o3 [ 12 ] demonstrated strong deep research capabilities by performing iterative chain-of-thought [ 22 , 8 ] generation, search, explanation, and analysis under a ReAct-style interaction loop. Motivated by these advances, a growing body of work has investigated search agents from multiple perspectives. For example, Tongyi DeepResearch [ 20 ] studies training paradigms and data construction for search agents, 14 including generating complex multi-hop questions from structured resources such as knowledge graphs, and improving agent performance via incremental pretraining or supervised fine-tuning on high-quality trajectories. Other lines of research leverage reinforcement learning to enhance search-agent capabilities, reducing the reliance on expensive trajectory collection. Recent systems such as WebWatcher [ 6] and MindWatcher [ 3] further extend open-source search agents to multimodal settings by incorporating image search APIs or local vision tools [ 15 ], enabling agents to handle multimodal real-time and long-tail problems. 

5.2 Search Agent Benchmarks 

Early evaluations of search agents often relied on conventional static QA datasets such as Hot-potQA [ 29 ] and SimpleQA [ 23 ]. When used to assess search-enabled agents, these benchmarks frequently suffer from severe data leakage [ 2], as answers (or near-answers) can be retrieved directly from the web or memorized by models during pretraining, undermining their validity as measures of agentic search. More recently, a number of benchmarks have been constructed in the live web setting to target multi-hop search, such as BrowseComp [ 24 ], WebWalkerQA [ 26 ], and WebShaper [ 18 ]. These benchmarks typically generate complex deep-search questions by leveraging hyperlink structures derived from Wikipedia or by constructing proprietary knowledge graphs that induce multi-step navigation and evidence aggregation. However, despite increasing task difficulty, they largely preserve the same evaluation paradigm: benchmarks grounded in real-world internet content remain inherently vulnerable to long-term issues, including knowledge leakage, dynamic obsolescence, and ground-truth drift as the web evolves. In contrast, MPW introduces the notion of evaluating search agents in a parallel world . By construct-ing a dynamic environment that is cognitively isolated from the agent and strictly constrained by scenario-specific atomic facts, MPW mitigates three fundamental challenges of web-based evaluation: data contamination, temporal invalidation of ground truth, and the lack of environmental control and reproducibility. 

## 6 Conclusion 

This paper presents the MPW framework and its accompanying benchmark, MPW-Bench, for systematically evaluating the end-to-end capabilities of Search Agents in a controlled, reproducible, and knowledge-isolated setting—a "parallel world" whose facts are provably unseen during model pre-training. MPW grounds its scenarios and question formats in real-world entities while strictly constraining world states and retrievable evidence through atomic facts (ParaWorld Laws). Under a unified web_search interaction interface, the PEM generates simulated SERPs together with fine-grained hit logs. The core isolation mechanism projects all critical facts onto a future timeline governed by these parallel-world laws, placing them beyond every evaluated model’s knowledge cutoff and thereby enforcing a must-search solving path. This design affords precise control over evidence provenance, retrievability, and evaluation specification, substantially mitigating the long-standing problems of data contamination, temporal staleness, and environmental uncontrollability inherent in live-Internet evaluations, while simultaneously providing the foundation for interpretable, process-level analysis. Experiments confirm that MPW-Bench offers a controlled, interpretable, and reproducible evaluation platform for Search Agents. More importantly, it clearly exposes the core shortcomings of current methods on complex tasks: (i) how to reliably formulate atomic queries that are both retrievable and coverage-complete, and (ii) when to continue searching versus when to stop and synthesize. We envision this benchmark catalyzing substantive advances in query formulation, coverage-driven interaction strategies, and evidence integration, ultimately enhancing the robustness and practical utility of Search Agents in real-world deployments.

16 A.1 Evaluation Settings 

To better characterize the capability bottlenecks of Search Agents in deep-search scenarios and improve the interpretability of our evaluation, we evaluate MPW-Bench under three progressive settings (Settings A/B/C). All three settings share the same dataset, the same PEM, and the same LLM-as-Judge protocol; they differ only in the information provided to the agent and the interaction constraints. Accordingly, they correspond to (i) an evidence-synthesis upper bound, (ii) guided search with explicit query-construction guidance, and (iii) fully end-to-end search without such guidance. 

Setting A: Oracle-Facts QA (evidence-synthesis upper bound). To estimate an achievable upper bound and verify that the benchmark instances are solvable, we define an oracle-facts setting in which the model is given the original question together with the full set of atomic facts for that instance (including any required computation/convention rules), and tool use is disabled. The model therefore does not perform retrieval; it only needs to correctly reason over and synthesize the provided evidence to produce the final answer. This setting primarily measures information integration and final synthesis under complete evidence, serving as an upper-bound reference for the downstream search settings once all necessary information has been obtained. 

Setting B: Guided Search (guided query formulation). To factor out the confounding effect of query formulation quality, we introduce a guided-search setting that retains the end-to-end interaction form: the model must still decompose the task and interact with PEM via web_search to gather evidence, but we explicitly guide it to issue hittable atomic queries (e.g., a single entity with a single attribute), and provide negative examples of compound queries together with correct decompositions. In this setting, the agent is evaluated primarily on its ability to decompose the problem and achieve sufficient evidence coverage, under assisted query construction. We further consider two guidance strengths within Setting B: (B1) a lightweight Guidance Prompt that provides concise rules and a few illustrative decomposition examples; and (B2) a stronger Few-shot Decomposition Prompt that additionally includes multiple case-level solution plans as references. Apart from the system prompt, all other evaluation components remain identical. 

Setting C: End-to-End ReAct Search (default). In the fully end-to-end setting, the model receives only the original question and basic tool instructions, without any additional guidance about how to construct effective queries. The model must independently (i) decompose the problem, (ii) formulate search queries, (iii) iteratively infer and refine what constitutes a high-quality query from interaction feedback, (iv) retrieve sufficiently comprehensive evidence, and (v) synthesize the final answer. This setting most closely reflects real-world usage of search agents and is the default configuration reported in our main results. These three settings decompose end-to-end performance into different capability components. Setting A reflects the upper bound of reasoning and synthesis when all required evidence is available; Setting B probes decomposition and evidence coverage under assisted query formulation; and Setting C measures the full stack of agentic behavior, including discovering effective querying strategies through interaction. This decomposition provides a principled explanation for low absolute end-to-end accuracy: if Setting A achieves high accuracy while Setting C remains low, then the primary bottleneck lies in retrieval and evidence coverage rather than ill-posed questions; moreover, the gap between Settings B and C quantifies the impact of query-construction guidance on end-to-end performance. ## B Asynchronous Inference Infrastructure for MPW 

MPW-Bench evaluation involves multi-turn interactive reasoning: each instance goes through repeated cycles of “thinking → tool calling → environment feedback.” Under sample-level synchronous inference, the system often leaves resources idle while waiting for tool returns or model responses from a single sample; this effect is amplified when interaction lengths vary across samples, and synchronous rollouts within a batch can incur substantial long-tail latency. To improve throughput for large-scale evaluation while keeping full trajectories auditable, we design an asynchronous inference engine tailored for search agents, whose core is fine-grained turn-level scheduling. 17 Turn-level concurrency and global scheduling. Our basic scheduling unit is not a “sample,” but a single ReAct turn. The system maintains an explicit state object SampleState for each sample, and schedules the next turn of all samples as asynchronous tasks. The scheduler uses asyncio to manage a large number of concurrent I/O requests, and controls the number of in-flight turns via a Semaphore (max_concurrent_turns ), enabling fine-grained dynamic load balancing. When a sample is blocked waiting for an LLM response or a tool return, the scheduler can immediately switch to execute turns from other samples, avoiding the common issues of “long samples slowing down the whole batch” and GPU underutilization in sample-level synchronous execution. Moreover, turn-level concurrency provides a sufficiently rich stream of concurrent requests for backends such as vLLM that support continuous batching, making it easier to form stable batches and thereby improving overall throughput and reducing tail latency. 

Explicit state machine and termination. Each sample follows an explicit state machine during inference ( running/finished/max_turns_reached/api_error/empty_response ). The exe-cution of a single turn proceeds as follows: we first call the LLM to generate the current action. If the output contains <answer> , the rollout terminates immediately, and the sample is marked as 

finished . If the output contains <tool_call> , the system executes the tool asynchronously (in our setting, web_search interacting with PEM) and feeds the tool output back as the next-turn observation. If neither an answer nor a tool call is produced, we inject a lightweight user reminder to encourage the model to continue in the next turn. When the number of assistant turns reaches the preset maximum, the sample is marked as max_turns_reached and terminated. 

Dual-track message storage and context truncation. Multi-turn interaction quickly increases context length; in search-agent settings, models may explore for many turns. To balance “runnability” and “evaluability,” we maintain two message tracks in SampleState : one is a context window used for actual inference. When the context length exceeds a predefined limit, we apply a sliding-window truncation strategy that drops early turns and retains only the system prompt, recent dialogue turns, and tool outputs. The other track preserves the full dialogue trajectory without truncation, which is used for offline evaluation, failure diagnosis, and behavioral analysis. This dual-track design allows inference to proceed under context constraints while retaining complete, auditable trajectories.
