## 摘要
我们提出了经验证的多智能体编排（Verified Multi-Agent Orchestration, VMAO），这是一个通过验证驱动的迭代循环来协调专用 LLM 智能体的框架。针对复杂查询，我们的系统将其分解为子问题的有向无环图（DAG），通过特定领域的智能体并行执行，利用基于 LLM 的评估来验证结果的完整性，并进行自适应重规划以弥补差距。主要贡献包括：(1) 在子问题 DAG 上实现具有自动上下文传播的依赖感知并行执行；(2) 验证驱动的自适应重规划，将基于 LLM 的验证器作为编排层级的协调信号；以及 (3) 可配置的停止条件，用于平衡回答质量与资源消耗。在 25 个专家精选的市场研究查询中，与单智能体基准相比，VMAO 将回答完整性从 3.1 提升至 4.2，来源质量从 2.6 提升至 4.1（1-5 分制），证明了编排层级的验证是多智能体质量保证的有效机制。

## Abstract
We present Verified Multi-Agent Orchestration (VMAO), a framework that coordinates specialized LLM-based agents through a verification-driven iterative loop. Given a complex query, our system decomposes it into a directed acyclic graph (DAG) of sub-questions, executes them through domain-specific agents in parallel, verifies result completeness via LLM-based evaluation, and adaptively replans to address gaps. The key contributions are: (1) dependency-aware parallel execution over a DAG of sub-questions with automatic context propagation, (2) verification-driven adaptive replanning that uses an LLM-based verifier as an orchestration-level coordination signal, and (3) configurable stop conditions that balance answer quality against resource usage. On 25 expert-curated market research queries, VMAO improves answer completeness from 3.1 to 4.2 and source quality from 2.6 to 4.1 (1-5 scale) compared to a single-agent baseline, demonstrating that orchestration-level verification is an effective mechanism for multi-agent quality assurance.

*   **核心问题**：现有的多智能体系统（如 AutoGen、MetaGPT）在处理需要跨领域专业知识和异构数据源的复杂查询时，缺乏有效的**编排层级验证**和**自适应调整机制**。
*   **研究动机**：在市场调研等领域，分析师需要从内部数据库、新闻、财务报告等多种渠道获取信息并进行综合。现有的方法（如辩论式或角色扮演式）虽然提升了推理能力，但无法确保结果的完整性，也无法在发现信息缺失时自动重新规划任务。

### 2. 方法论
VMAO 框架采用了 **“计划-执行-验证-重规划-综合” (Plan-Execute-Verify-Replan-Synthesize)** 的五阶段循环：
*   **计划 (Plan)**：将复杂查询分解为子问题的**有向无环图 (DAG)**。每个子问题包含 ID、代理类型、依赖关系、优先级和验证标准。
*   **执行 (Execute)**：根据 DAG 的依赖关系并行执行任务。利用自动上下文传播，将上游结果作为下游任务的输入。
*   **验证 (Verify)**：使用基于 LLM 的验证器评估每个子问题的结果，生成完整性评分（0-1）并识别缺失方面或矛盾点。
*   **重规划 (Replan)**：根据验证反馈，决定是重试失败的任务、生成新的子问题以填补空白，还是合并现有结果。
*   **综合 (Synthesize)**：当满足停止条件（如完整性 > 80%、达到最大迭代次数或 Token 预算）时，进行分层综合，生成带有准确来源引用的最终报告。
*   **智能体分类**：将智能体分为三层：数据层（RAG、Web、财务）、分析层（推理、原始数据处理）和输出层（文档、可视化）。

### 6. 主要结论与发现
*   **性能提升**：VMAO 在完整性上从 3.1 提升至 **4.2**，在来源质量上从 2.6 提升至 **4.1**。
*   **任务类型差异**：在开放式的“战略评估”任务中提升最显著（+53%），因为这类任务最需要动态调整计划；而在定义明确的“性能分析”任务中提升较小。
*   **验证的价值**：编排层级的验证是多智能体系统质量保证的有效信号，能显著减少因工具调用失败或搜索结果不足导致的遗漏。

### 7. 优点
*   **结构化编排**：通过 DAG 明确了任务依赖，支持高效的并行执行。
*   **闭环反馈**：引入了显式的验证和重规划步骤，使系统具备了“自我纠错”能力。
*   **模块化设计**：采用模型上下文协议 (MCP) 独立运行工具服务，易于扩展和维护。
*   **可配置性**：提供了灵活的停止条件（如 Token 预算、迭代上限），便于平衡成本与质量。

### 8. 不足与局限
*   **成本与延迟**：Token 消耗是单智能体的 8.5 倍，响应时间也大幅增加，不适合对实时性要求高的场景。
*   **验证局限**：验证器主要评估“完整性”而非“绝对事实性”，可能无法完全排除细微的事实错误或幻觉。
*   **模型依赖**：实验仅基于 Claude 系列模型，在其他模型家族上的泛化能力尚未验证。
*   **样本量限制**：25 个查询的评估集规模相对较小，缺乏更广泛的统计显著性分析。
