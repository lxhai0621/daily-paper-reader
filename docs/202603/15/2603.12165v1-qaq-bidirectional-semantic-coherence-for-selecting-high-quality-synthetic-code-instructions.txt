Abstract 

Synthetic data has become essential for training code generation models, yet it introduces signif-icant noise and hallucinations that are difficult to detect with current metrics. Existing data se-lection methods like Instruction-Following Dif-ficulty (IFD) typically assess how hard a model generates an answer given a query ( A|Q). How-ever, this metric is ambiguous on noisy syn-thetic data, where low probability can distin-guish between intrinsic task complexity and model-generated hallucinations. Here, we pro-pose QAQ, a novel data selection framework that evaluates data quality from the reverse di-rection: how well can the answer predict the query ( Q|A)? We define Reverse Mutual Infor-mation (RMI) to quantify the information gain about the query conditioned on the answer. Our analyses reveal that both extremes of RMI sig-nal quality issues: low RMI indicates semantic misalignment, while excessively high RMI may contain defect patterns that LLMs easily rec-ognize. Furthermore, we introduce a selection strategy based on the disagreement between strong and weak models to identify samples that are valid yet challenging. Experiments on the WarriorCoder dataset demonstrate that se-lecting just 25% of data using stratified RMI achieves comparable performance to full-data training, significantly outperforming existing data selection methods. Our approach high-lights the importance of bidirectional semantic coherence in synthetic data curation, offering a scalable pathway to reduce computational costs without sacrificing model capability.

containing fabricated terminology (e.g., “convert a gritty sulla matrix”), where models generate syntac-tically valid code that echoes these made-up terms, producing high surface-level quality scores while providing little learning signal; (2) off-topic queries that should not appear in a code dataset at all (e.g., a greeting “Hi!”), paired with incoherent responses (e.g., starting with “compensated...”)—indicating a completely broken Q-A pair. Notably, methods like WarriorCoder focus on an-swer quality : multiple models generate competing solutions, and the highest-scored answer is selected for each query. However, this pipeline largely over-looks query quality . A poorly-specified or seman-tically incoherent query, even when paired with a technically correct answer, provides limited learn-ing signal. Existing data selection methods like Instruction-Following Difficulty (IFD) (Li et al., 2024b) op-erate in the A|Q direction. Although mutual in-formation is theoretically symmetric, due to the autoregressive nature of language models, its prac-tical estimation via conditional perplexity is not necessarily consistent. Empirically, we find low correlation ( ρ = 0 .252 ) between the two, confirm-ing they capture distinct aspects of data quality. We propose QAQ (Query, Answer-Query), a method that measures quality from the reverse di-rection: PPL (Q|A), which captures how well the answer explains the question. The intuition is anal-ogous to a student who struggles to understand a problem, but finds clarity after seeing the solution. We define RMI as log PPL (Q) − log PPL (Q|A),which captures the information gain that the an-swer provides about the question: • Low RMI : The answer provides little infor-mation gain about the question; seeing the answer does not help predict the question. • Very high RMI : The answer makes the query trivially predictable, often due to superficial or shortcut patterns such as paraphrasing or repetition. Furthermore, we observe that strong and weak models disagree on sample quality. Samples where only the strong model recognizes high RMI may contain sophisticated patterns that weak models cannot capture. We call this the “cognitive gap”. Selecting samples with high cognitive gap yields better training signal than selecting samples where both models agree. Our contributions are: • We identify that seedless synthetic data pipelines overlook query quality, and propose QAQ, a data selection method based on Re-verse Mutual Information (RMI) that captures query-answer semantic alignment. • We empirically show that RMI serves as an effective quality indicator, where both mis-aligned and shortcut-dominated samples can be filtered through its value distribution. • We introduce model disagreement as a selec-tion signal, capturing samples with high “cog-nitive gap” between strong and weak models. • We demonstrate that 25% of data selected by our method achieves comparable performance to full-data training while outperforming ex-isting data selection methods. 

2 Related Work 

Data Selection for Instruction Tuning. Several approaches have been proposed to select high-quality samples from instruction-tuning datasets. IFD (Li et al., 2024b) measures sample quality us-ing PPL (A|Q)/PPL (A), the ratio of conditional to unconditional answer perplexity. Samples with IFD close to (but less than) 1 are preferred: IFD ≪ 1

suggests the answer is trivially predictable from the instruction, while IFD > 1 indicates the instruction actually hinders answer generation. Superfilter-ing (Li et al., 2024a) shows that weak models can approximate strong models’ IFD scores, enabling efficient filtering at scale. These methods operate in the A|Q direction, measuring generation difficulty rather than query-answer semantic alignment. Beyond perplexity-based methods, gradient-based approaches like LESS (Xia et al., 2024) use influence estimation to identify training samples most relevant to target tasks. Multi-dimensional methods like D3 (Zhang et al., 2025) jointly op-timize diversity, difficulty, and dependability via coreset objectives. Recent work explores orthog-onal selection criteria: T-SHIRT (Fu et al., 2025) improves IFD through token-level selection and neighborhood robustness; SCAR (Li et al., 2025) targets style consistency between instructions and responses; ProDS (Guo et al., 2025) aligns selec-tion with task-specific preferences via preference Figure 2: Overall workflow of the QAQ framework for synthetic code data selection. optimization. These methods optimize for diversity, style, or preference alignment, but do not directly assess query-answer semantic coherence. 

Synthetic Data for Code Generation. High-quality datasets are essential for code generation. Code Alpaca (Chaudhary, 2023) and WizardCoder (Luo et al., 2024) use Evol-Instruct to evolve seed instructions into more complex programming prob-lems. OSS-Instruct (Wei et al., 2023) generates diverse problems by extracting coding concepts from open-source code snippets. These seed-based methods inherit biases from their curated seeds, constraining diversity to the patterns present in the original data. In contrast, seedless approaches like Magpie (Xu et al., 2024) and WarriorCoder (Feng et al., 2025) achieve greater diversity by prompting aligned LLMs directly without relying on human-curated data. However, this freedom comes at a cost: without seed constraints, the generated data contains more noise and semantic inconsistencies. While WarriorCoder selects the highest-scored an-swer for each query through cross-model evalu-ation, query quality itself remains largely unad-dressed. 

Mutual Information in NLP. Pointwise mutual information (PMI) has been widely used in NLP for measuring word associations and semantic similar-ity (Church and Hanks, 1990). At the token level, PMI helps identify collocations and construct se-mantic spaces. This concept can be extended to the sequence level, where mutual information between query and answer sequences reflects their semantic coherence. 

Model Comparison for Data Quality. Using multiple models to assess data has proven effective in various contexts. Helm et al. (2025) compare short-context and long-context models to weight tokens for long-range language modeling. Rho-1 (Lin et al., 2024) compares trained and untrained versions of the same model to identify tokens with high “excess loss” for selective pre-training. These works focus on token-level selection, while sample-level selection using model disagreement remains underexplored. 

3 QAQ: Reverse Mutual Information for Data Selection 

Given a large-scale instruction-tuning dataset D =

{(Qi, A i)}Ni=1 for code generation, our goal is to se-lect a high-quality subset S ⊂ D that achieves com-parable performance to training on the full dataset while significantly reducing computational costs. 

3.1 Reverse Mutual Information 

Prior work such as IFD (Li et al., 2024b) captures whether the instruction facilitates answer genera-tion. Samples with IFD close to (but less than) 1 are preferred, representing what the model “does not yet know”. We propose an alternative perspective: instead of measuring generation difficulty, we measure how well the answer explains the question. Specifically, we compute the Reverse Mutual Information (RMI) as: RMI (Q, A ) = log PPL (Q) − log PPL (Q|A) (1) where PPL (Q) is the perplexity of the question alone, and PPL (Q|A) is the perplexity of the ques-tion conditioned on the answer. Comparison with IFD. While IFD operates in the A|Q direction, measuring “instruction-following difficulty”, our approach operates in the Q|A direction, measuring “answer explanatory power”. We argue that a good code solution should clearly reflect the original problem specification: given the code, one should be able to infer what problem it solves.

3.2 Stratified RMI 

Directly filtering by global RMI rankings intro-duces a confounding factor: question complex-ity. Simple questions naturally have low PPL (Q),leading to lower RMI scores regardless of answer quality. As shown in Figure 4, the relationship between RMI and log PPL (Q) exhibits clear het-eroscedasticity: samples with low PPL (Q) have narrowly distributed RMI values (concentrated at lower ranges), while samples with high PPL (Q)

show much greater RMI variance. This indicates that RMI and question complexity capture different aspects of data quality. Without stratification, sim-ple but valid questions would be incorrectly filtered out due to their inherently lower RMI ceiling. To address this, we propose Stratified RMI : we first partition the dataset into K = 10 bins (deciles) based on PPL (Q), then compute RMI rankings in-dependently within each bin. We use K = 10 as a simple default without extensive tuning, as our goal is to demonstrate the effectiveness of stratification rather than to optimize this hyperparameter. This ensures that samples are compared against others of similar question complexity. Formally, let Bk denote the k-th bin containing samples with similar PPL (Q) values. 3.3 Model Disagreement 

Following prior work showing that weak models can approximate strong models’ data quality scores (Li et al., 2024a), we first evaluate whether RMI ex-hibits similar model-invariance. While strong and weak models show high rank correlation (Spear-man ρ ≈ 0.7) and 76.1% overlap in top-50% selec-tion, their downstream performance differs substan-tially (73.17 vs 68.90 on HumanEval+, Table 1). This suggests the 24% non-overlapping samples are disproportionately important. We therefore explore model disagreement as an alternative signal. We compute stratified RMI ranks using two models: • A strong model Ms (e.g., DeepSeek-Coder-6.7B-Base) • A weak model Mw (e.g., Qwen3-0.6B) For each sample, we obtain two normalized ranks rs, r w ∈ [0 , 1] and define: Diff = rs − rw (disagreement signal) (4) High Diff indicates samples where the strong model gives high RMI rank while the weak model gives low RMI rank. We hypothesize that such sam-ples are both valid (recognized by strong model) and learnable (challenging for weak model). We validate this hypothesis empirically in Section 4.3. Our final selection filters samples with Diff > τ ,where we set τ = 0 .1 (approximately 25% of sam-ples). 

4 Experiments 

4.1 Experimental Setup Dataset. We use a reproduction of the Warrior-Coder dataset (Feng et al., 2025) from Hugging-Face, containing 329K instruction-response pairs for code generation created through multi-agent collaboration. After filtering samples exceeding 2048 tokens, approximately 310K samples remain. The original dataset is not publicly available. 

Models. For computing RMI scores, we use: • Strong model: DeepSeek-Coder-6.7B-Base (Guo et al., 2024) • Weak model: Qwen3-0.6B (Team, 2025) For fine-tuning evaluation, we train DeepSeek-Coder-6.7B-Base on selected subsets. 

Training Details. We fine-tune for 3 epochs us-ing LlamaFactory (Zheng et al., 2024). Due to the noisy nature of our dataset, we refrain from ex-cessively reducing the batch size. Batch sizes and learning rates are adjusted by dataset size: for full data, we use batch size 512 and learning rate 1.2e-4; for 50% data, batch size 256 and learning rate 0.8e-4; for 25% data, batch size 256 and learning rate 0.4e-4. Additionally, the learning rate sched-uler follows a cosine decay with a warmup ratio of 0.2. 

Evaluation. We evaluate on HumanEval (Chen et al., 2021), HumanEval+, MBPP (Austin et al., 2021), and MBPP+ benchmarks (Liu et al., 2023) using greedy decoding. 

Baselines. 4.2 Main Results 

We fine-tune DeepSeek-Coder-6.7B-Base on se-lected subsets from WarriorCoder (310K samples) and evaluate on HumanEval(+) and MBPP(+) (Liu et al., 2023). Results are shown in Table 1. Overall, stratified RMI selection with 50% data matches or exceeds full-data performance, while the mid-range 25% subset (RMI 50-75%) achieves comparable results. Both extremes (top-25% and bottom-25%) underperform, and RMI-based se-lection consistently outperforms baseline methods. Key findings: • RMI outperforms most baselines : At 25% data, RMI 50-75% outperforms Random, IFD, and SCAR on both benchmarks. It achieves results comparable to RDS+, surpassing it on HumanEval+ (72.56 vs 71.34) while remain-ing competitive on MBPP+. • Middle range is optimal : RMI 50-75% (25% data) matches full-data performance on HumanEval+ (72.56) and approaches it on MBPP+ (58.47 vs 59.52). This “sweet spot” contains samples with good semantic align-ment without trivial paraphrasing. • Both extremes are harmful : RMI Top-25% underperforms full data by 2.4/2.4 points on HumanEval+/MBPP+ (70.12/57.14 vs 72.56/59.52), suggesting extremely high RMI indicates certain defect patterns that language models easily recognize, such as answer para-phrasing the query. RMI Bottom-25% un-derperforms by 4.9/4.8 points (67.68/54.76), confirming low RMI indicates semantic mis-alignment. 

Case Study: Defect Patterns in High RMI. To illustrate why extremely high RMI samples are harmful, we examine an example from the top RMI percentile (RMI = 3 .37 ):              

> Query :How to convert a gritty sulla matrix with floating discharges into a fairly crisp supra array pre-dominantly within specific thresholds using Python?
> Answer : Converting a Gritty Sulla Matrix ... To con-vert a gritty sulla matrix with floating discharges into a fairly crisp supra array ... (followed by generic NumPy thresholding code)

The query contains nonsensical jargon, and the an-swer simply paraphrases these terms (underlined). The answer makes the query highly predictable (hence high RMI) but provides no genuine learning signal. This illustrates one type of defect pattern that concentrates in high-RMI regions: answers echo query keywords, reducing PPL (Q|A) without semantic substance. 

Case Study: Low RMI as Misalignment. Con-versely, we examine a sample with negative RMI (RMI = −0.13 ): 

> Query : Hi!
> Answer : compensated, let’s get you the help you need. Could you please provide more details about what you’re working on or what specific Python cod-ing question you have? I’m here to assist you with any Python-related challenges you might be facing. Whether it’s a beginner question, a complex algorithm, or anything in between, feel free to share the details, and I’ll do my best to help!

The query “Hi!” is not a meaningful instruction for a code dataset. Consequently, the generated answer—despite attempting to be helpful—cannot provide information that helps predict the query. This is a typical example of semantic misalign-ment : a poorly-specified query leads to an answer that has little semantic connection to the query con-tent (hence negative RMI). Such samples should be filtered out as they provide poor learning signal for code generation.

Ablation: Disagreement vs. Consensus. To val-idate the effectiveness of our disagreement-based selection, we compare four selection strategies us-ing fixed hyperparameters (batch size 256, learning rate 4e-5, warmup ratio 0.2). We report results from the best checkpoint for each method. Results are shown in Table 3. Key observations: (1) Diff > Sum : Diff-High outperforms Sum-High by 3 points on HumanEval+ (71.95 vs 68.90), validating that disagreement sig-nals are more informative than consensus. (2) 

Sum-Low is worst : Samples where both models agree on low RMI yield the poorest results, con-firming they should be filtered out. (3) Diff-High vs Diff-Low : Diff-High favors HumanEval while Diff-Low favors MBPP. One possible explanation is that strong models identify complex patterns ben-eficial for challenging tasks, while weak models recognize simpler patterns beneficial for straight-forward tasks. 

Stability of RMI Across Training. To verify that RMI captures intrinsic data properties, we compute RMI using the same model before and after fine-tuning. The Pearson correlation be-tween pre-training and post-training RMI scores is ρ = 0 .9539 , indicating that RMI rankings are highly stable across training. This makes RMI suitable for static one-time selection without re-quiring iterative re-scoring. However, the moderate correlation between strong and weak model rank-ings ( ρ = 0 .70 ) suggests that RMI also reflects model-specific knowledge: stronger models rec-ognize quality patterns that weaker models miss. This dual nature of RMI (stable within a model, yet sensitive to model capability) is precisely what enables the Diff signal to capture “cognitive gap” samples. 

Sample Overlap Analysis. We analyze whether disagreement-based and consensus-based selection identify the same samples. Specifically, we com-pare two 25% selection strategies: (1) Diff-High: samples with high rs −rw, and (2) Sum-High: sam-ples with high rs+rw (“double-high”). The overlap between these two selected sets is only 13.85% ,indicating that they capture fundamentally different sample populations. This low overlap explains the performance gap in Table 3: Diff-High and Sum-High achieve 71.95 vs. 68.90 on HumanEval+. We hypothesize that Sum-High samples include many paraphrasing 

cases, where the answer trivially echoes the ques-tion. Repetition is a common phenomenon in neu-ral text generation (Holtzman et al., 2020): models tend to repeat input phrases verbatim. Crucially, such repetition is a simple pattern that both strong and weak models can easily detect, leading to high RMI scores from both ( rs high, rw high, hence Sum high but Diff low). By selecting Diff-High samples, we naturally filter out these defect pat-terns and retain samples with genuine semantic content that only strong models recognize. Additionally, we compare the overlap between strong and weak model top-50% selections (with-out using the Diff signal). The overlap is 76.1%, substantially higher than the 13.85% above. De-spite this high overlap, the two selections yield different downstream performance (Table 1), con-firming that the remaining 23.9% non-overlapping samples carry significant training signal. The dis-agreement signal captures this unique subset that neither model alone would prioritize. 

5 Conclusion 

We proposed a novel data selection framework for code generation based on reverse mutual informa-tion and model disagreement. Our key insight is that measuring “answer explanatory power” via PPL (Q|A) provides a different and complementary signal to traditional “generation difficulty” mea-sured by PPL (A|Q). Furthermore, samples with high cognitive gap, where strong models recog-nize quality that weak models miss, are particularly valuable for training. Our experiments reveal three key findings: (1) RMI-based selection with stratification consistently outperforms IFD-based selection, validating the im-portance of reverse direction analysis; (2) RMI out-liers at both ends correlate with undesirable sam-ples; (3) Model disagreement captures a unique subset of valuable samples (only 13.85% overlap with consensus-based selection) that significantly improves training efficiency. Practically, our method achieves comparable per-formance to full-data training using only 25% of the data, demonstrating that careful data selection can substantially reduce computational costs with-out sacrificing model capability. Looking forward, our framework opens several research directions: extending the approach to other domains beyond code generation, exploring alternative model pairs for disagreement signals, and investigating curriculum learning strategies that dynamically adjust selection criteria during training. 

Limitations 

Our work has several limitations that future re-search should address: Single Dataset. Our experiments are conducted exclusively on the WarriorCoder dataset for code generation. While WarriorCoder represents state-of-the-art seedless synthetic data generation, it may have specific noise patterns that differ from other datasets. Future work should validate the approach on diverse datasets including other code generation corpora (e.g., Evol-Instruct), general instruction-tuning datasets (e.g., Alpaca, WizardLM), and domain-specific datasets. 

Model Selection Sensitivity. The effectiveness of the disagreement signal depends on the choice of strong and weak models. We use DeepSeek-Coder-6.7B-Base and Qwen3-0.6B based on their avail-ability and capability gap, but the optimal model pair may vary across domains. A systematic study of how model family, size, and training data affect the disagreement signal is needed. 

Hyperparameter Tuning. We use fixed hyper-parameters throughout ( K = 10 bins, τ = 0 .1

threshold) without extensive tuning. While our ex-periments demonstrate robustness across different selection ratios, more fine-grained hyperparameter search may further improve performance. 

Reproducibility. Due to time constraints, we do not report results across multiple random seeds. While the large dataset size (310K samples) pro-vides some stability, future work should include confidence intervals. 

Computational Cost. Computing RMI scores re-quires two forward passes per sample (for PPL (Q)

and PPL (Q|A)), which adds overhead compared to standard training. However, this is a one-time cost that enables significant training data reduction.

Implementation Note In both cases, PPL (Q|A)

is calculated by performing a forward pass on the ground-truth question tokens represented by [Q] in the Assistant turn, while teacher-forcing the System and User inputs as context. 

B Sensitivity to Chat Template and Task Framing 

We investigate how specific task instructions and formatting affect RMI and IFD signals. 

Standard Implementations (Template-based). 

In our main experiments, both metrics utilize struc-tured chat templates. For **Standard RMI**, the reverse generation task is highly constrained: 

“TASK: Given an answer, generate the most likely computer science question that this answer is responding to. If the inferred question is outside computer science, respond with ‘INVALID’ .” 

For **Template-based IFD**, we compute the perplexity of the answer A within the following structured templates:    

> System : Standard system prompt
> User : [None]
> Assistant :[A]
> System : Standard system prompt
> User :[Q]
> Assistant :[A]

Variations for Sensitivity Analysis. To isolate the impact of task framing, we compare our stan-dard method against several variations: • Raw IFD : Following the original implementa-tion proposed by Li et al. (2024b), Raw IFD is computed via simple text concatenation: [Q] + [A] for the conditional part, and [A] for the unconditional part, without any system prompts or role indicators (User/Assistant). • Simplified Task Prompt ( ρ = 0 .88 ): Re-moves both the domain specificity and the neg-ative constraint, using a more general instruc-tion: “TASK: Given an answer, generate the most likely question that this answer is responding to.” 

• System Context Only ( ρ = 0.79 ): For 

P P L (Q), we retain the System Prompt but re-move the explicit “TASK” instruction . This tests whether the model’s persona alone (e.g., “You are an AI assistant”) is sufficient for esti-mating query probability. • Prefix-only ( ρ = 0.70 ): For P P L (Q),we strip away the System Prompt and use only a minimal newline-delimited indicator ("Question:" ). • Pure Raw ( ρ = 0 .62 ): For P P L (Q), we calculate the perplexity from the very first token of the query with zero context (cold start). 

Results and Analysis. As shown in Table 4, IFD is remarkably stable ( ρ = 0 .957 ) even under raw concatenation. In contrast, RMI drops as we strip away the template and context. The correlation decreases from 0.88 (Simplified Template) to 0.79 (System-Prefix), 0.70 (Prefix-only), and 0.62 (Pure Raw). This confirms that while the specific word-ing of the template matters less, the presence of suf-ficient context for P P L (Q) is critical. Even with-out a task template, providing the system prompt (ρ = 0 .79 ) significantly stabilizes the metric com-pared to raw inputs.

The primary source of instability is the contex-tual imbalance in P P L (Q) calculation . In the 

Pure Raw setting, the model must predict the query from a "cold start" without knowing it is handling a CS query. Since queries are typically short, the high loss incurred by the initial tokens dominates the average P P L (Q), biasing the RM I upwards and introducing noise (see Figure 5). 

Discussion: Why IFD is More Robust. IFD re-mains robust because P P L (A) is calculated on an-swer sequences (code) that are typically longer and follow familiar structural patterns ( def , import ). Moreover, A is always conditioned on Q in the IFD calculation, avoiding the "zero context" problem.
